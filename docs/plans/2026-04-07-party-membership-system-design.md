# 入党文档管理系统 - 设计文档

## 1. 项目概述

### 1.1 背景
重建入党文档管理系统，用于管理入党申请人从申请到转正的完整流程（5阶段26步骤），以及相关文档管理。

### 1.2 目标用户
- **申请人 (applicant)**: 提交入党申请，查看进度，上传文档
- **支部书记 (secretary)**: 审批/驳回申请人上传的文档，补充上传
- **入党联系人 (contact)**: 跟踪申请人培养情况
- **党务工作者/管理员 (admin)**: 最终审批，管理用户/支部/模板

### 1.3 规模
- 多个党支部
- 总用户约 400 人

## 3. 数据模型

### 3.1 核心表

| 表名 | 用途 | 主要字段 |
|-----|------|---------|
| `users` | 用户 | username, name, role, branch_id |
| `branches` | 支部 | name, description |
| `step_definitions` | 26步定义 | step_code(L1-L26), stage(1-5), name |
| `applications` | 入党申请 | user_id, branch_id, current_stage, current_step, status |

| `documents` | 文档 | application_id, step_code, filename, file_path |
| `templates` | 模板 | name, step_code, filename, file_path |
| `quarterly_reviews` | 季度考察 | application_id, quarter, content |

### 3.2 用户角色

| 角色 | 代码 | 权限范围 |
|------|------|---------|
| 管理员 | `admin` | 全局管理，最终审批 |
| 支部书记 | `secretary` | 本支部申请管理，初审 |
| 入党联系人 | `contact` | 查看分配的申请人 |
| 申请人 | `applicant` | 查看自身申请，上传文档 |

## 5. 核心业务流程

### 5.1 文档审批流程

```
申请人上传文档 → 书记审批/补充 → 管理员审批 → 步骤完成
```

### 5.2 步骤状态

| 状态 | 说明 |
|------|------|
| `pending` | 待申请人上传 |
| `pending_secretary` | 待书记审批 |
| `rejected_by_secretary` | 书记驳回，待申请人重新上传 |
| `pending_admin` | 待管理员审批 |
| `rejected_by_admin` | 管理员驳回，待书记重新处理 |
| `completed` | 已完成 |

### 5.3 状态流转

```
[pending] ──申请人上传──▶ [pending_secretary]
                              │
                        书记驳回
                              │
                              ▼
                       [rejected_by_secretary]
                              │
                        申请人重新上传
                              │
                              ▼
                       [pending_secretary]
                              │
                         书记通过
                              │
                              ▼
                       [pending_admin]
                              │
                        管理员驳回
                              │
                              ▼
                       [rejected_by_admin]
                              │
                        书记重新处理
                              │
                              ▼
                       [pending_admin]
                              │
                         管理员通过
                              │
                              ▼
                       [completed]
```

### 5.4 各角色操作权限

| 状态 | 申请人 | 书记 | 管理员 |
|------|--------|------|--------|
| `pending` | 上传文档 | - | - |
| `pending_secretary` | 查看状态 | 通过/驳回/补充上传 | 查看 |
| `rejected_by_secretary` | 重新上传 | 查看 | 查看 |
| `pending_admin` | 查看状态 | 查看状态 | 通过/驳回 |
| `rejected_by_admin` | 查看状态 | 重新处理 | 查看 |
| `completed` | 查看 | 查看 | 查看 |

## 6. 通知系统

### 6.1 通知类型

| 触发事件 | 通知对象 | 内容 |
|---------|---------|------|
| 申请人上传文档 | 书记 | 有新文档待审批 |
| 书记驳回 | 申请人 | 文档被驳回，需重新上传 |
| 书记通过 | 管理员、申请人 | 步骤已初审通过 |
| 书记补充上传 | 管理员 | 有新文档待确认 |
| 管理员驳回 | 书记、申请人 | 步骤被驳回 |
| 管理员通过 | 书记、申请人 | 步骤已完成 |

### 6.2 通知存储

新建 `notifications` 表：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| user_id | INTEGER | 接收者 |
| title | VARCHAR(100) | 通知标题 |
| content | TEXT | 通知内容 |
| link | VARCHAR(200) | 跳转链接 |
| is_read | BOOLEAN | 是否已读 |
| created_at | DATETIME | 创建时间 |

## 7. 时间线视图

### 7.1 设计

申请人查看进度的核心视图，展示全部26步：

**图标说明：**
- ● 已完成步骤
- ◉ 当前步骤（进行中/审批中）
- ○ 未开始步骤

**状态标签：**
- ✓ 已完成
- ⏳ 审批中（显示当前审批人）
- 待处理

### 7.2 五阶段划分

| 阶段 | 名称 | 步骤 |
|------|------|------|
| 1 | 递交入党申请 | L1-L2 |
| 2 | 入党积极分子培养 | L3-L7 |
| 3 | 确定发展对象 | L8-L10 |
| 4 | 接收预备党员 | L11-L17 |
| 5 | 预备党员转正 | L18-L26 |

## 8. 前端设计

### 8.1 配色方案

| 元素 | 颜色 | 用途 |
|------|------|------|
| 主色 | `#C41E3A` (中国红) | 头部导航、按钮、强调元素 |
| 深红 | `#8B0000` | 悬停状态、进度条填充 |
| 金色 | `#D4AF37` | 党徽、图标点缀、成功状态 |
| 浅红 | `#FFF5F5` | 卡片背景、悬停背景 |
| 白色 | `#FFFFFF` | 内容区背景 |
| 灰色 | `#F5F5F5` | 页面背景、分隔线 |
| 深灰 | `#333333` | 正文文字 |

### 8.2 页面头部

- 左侧：党徽 + 系统名称
- 右侧：用户信息 + 退出按钮
- 背景：渐变红 → 深红

### 8.3 按钮样式

| 类型 | 样式 |
|------|------|
| 主要操作 | 红色背景 `#C41E3A`，白色文字 |
| 成功/通过 | 金色背景 `#D4AF37`，深色文字 |
| 危险/驳回 | 深红边框 `#8B0000`，红色文字 |
| 默认/取消 | 白色背景，灰色边框 |
